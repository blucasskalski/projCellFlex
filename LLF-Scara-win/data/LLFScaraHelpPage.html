LLF Scara Help Page


« LLF Cellflex Scara » est un logiciel de pilotage du poste D (Scara) du projet Cellflex du Lycée La Fayette (sections de STS ET et IRIS). Ce projet concerne l’assemblage automatique d’un puzzle de 20 pièces représentant le logotype de l’établissement.


Brève présentation du robot

Le robot Scara utilisé est un modèle 5 axes basé sur des éléments linéaires et rotatifs de type « PowerCube » reliés par bus CAN.



On considère le robot posé sur le plan orthonormé 0XY et en élévation suivant 0Z. La position (x,y,z) de l’organe terminal est définie par les positions angulaires des actuateurs corps et coude (coordonnées x et y), et par la position de l’axe linéaire vertical (coordonnée z).

Le robot est de morphologie RPRR (trois articulations rotoïdes et une articulation prismatique).

Les trois premiers d.d.l. (degré de liberté), ici RPR, constituent le « porteur du robot ». Les d.d.l. suivants sont relatifs au poignet (organe terminal).

Le rôle du porteur de robot est de fixer la position dans l’espace du point P d’intersection des axes du poignet.

Le point piloté dans l’espace est nommé TCP (Tool Center Point).

Dans notre cas, le TCP peut être arbitrairement défini à l’intersection de l’axe 0Z du poignet et du plan 0XY passant par la surface d’appui des doigts sur les pièces…

Démarrage du système

Les liaisons CAN et puissance du robot passent par l’armoire générale de pilotage du site. Le bus CAN doit être connecté au poste hôte du logiciel « LLF Cellflex Scara » via un câble spécifique de type CAN-USB (adaptateur ESD). Le pilote de cet adaptateur n’est disponible que pour Windows XP…


La mise sous tension de l’armoire générale provoque le démarrage du logiciel d’interface sur l’écran tactile de façade ; comme indiqué, un appui sur cet écran permet ensuite la mise sous tension et la mise en air des postes du processus. À ce stade, le robot Scara devient utilisable.


Interface utilisateur

Le logiciel « LLF Cellflex Scara » peut maintenant être démarré. Son interface utilisateur est décomposée en deux onglets principaux :

  • Manual Mode : cette page doit être utilisée en priorité afin d’établir la connexion avec le robot et la configuration de ses trajectoires. Elle offre par ailleurs un accès bas niveau de pilotage de chacun des axes du robot.

  • Automatic Mode : cette page est dédiée au processus Cellflex de constitution du puzzle, elle implémente le serveur Modbus TCP/IP et propose une visualisation graphique de l’état courant du puzzle.


Chacune des pages dispose d’une fenêtre de suivi (monitoring) où sont reportés les messages d’information et d’erreur.


Le logiciel est par défaut en mode simulation ; ce mode rend inopérant la page Manual Mode. Le mode simulation permet d’exploiter l’esclave Modbus sans présence obligatoire du robot ; dans ce cas, une petite animation graphique simule les déplacements de pièce entre les palettes et le puzzle.

Configuration du logiciel

  1. Charger la cartographie Modbus du poste (Automatic Mode, bouton « Load Modbus Configuration »). Le fichier à charger est nommé modbus_scara.csv ;

  2. Démarrer le service TCP/IP, par défaut sur le port 502 (Automatic Mode, bouton « Start ») ;

  3. Le serveur Modbus est maintenant démarré ; en cas d’échec avec la version actuelle du logiciel, un redémarrage de celui-ci est nécessaire pour permettre un nouveau chargement de fichier de configuration.

  4. Activer le Manual Mode (case à cocher en bas à gauche de l’IHM) ;

  5. Charger le fichier de configuration du robot (Manual Mode, bouton « Load »). Le fichier à charger est nommé cellflex_scara.csv ;

  6. Compléter si nécessaire la liste de points (Manual Mode, bouton « Calculate »).


Le chargement de la configuration du robot initialise les valeurs de vitesse et d’accélération des axes, la liste des points finaux des trajectoires (table Tool Center Point Waypoints) et les ordres de rotation de la pince nécessaires à l’orientation des pièces lors des transferts (table Assembly Mode rotations).

Chaque waypoint est constitué des coordonnées x, y, z du TCP et d’un correcteur éventuel de rotation de la pince :

  • Les points Pa_Mon et Pa_Dem correspondent respectivement aux positions de prise ou dépose des pièces sur les palettes en mode montage et démontage ;

  • Les points Pu_00 et Pu_19 sont les positions sur le puzzle pour les emplacements supérieur-gauche et inférieur-droit ; les autres points, de Pu_01 à Pu_18, peuvent être calculés automatiquement par la commande « Calculate » ;

  • Le point PWait permet de définir la position spatiale d’attente rejointe à la fin des cycles d’assemblage ou de désassemblage ;

  • Le point PTrash est à renseigner pour le rebut des éventuelles pièces non reconnues…

Initialisation du robot

  1. Activer le Manual Mode (case à cocher en bas à gauche de l’IHM) ;

  2. Établir la connexion CAN avec le robot (Page Manual Mode, bouton « Open Device ») ;

  3. Sélectionner tous les axes (Page Manual Mode, bouton « Select All ») ;

  4. Ré-initialiser les axes (Page Manual Mode, bouton « Reset ») ;

  5. Lancer la procédure de Prise d’Origine Machine (Page Manual Mode, bouton « Home ») ;

  6. Le robot est maintenant prêt à effectuer des trajectoires…


Mise en exploitation

Les étapes ci-dessus de démarrage du service TCP/IP et d’initialisation du robot sont nécessaires et suffisantes. La page Automatic Mode permet de suivre le déroulement des opérations ; en cas de défaut de fonctionnement, des messages liés au pilotage proprement dit du robot peuvent être reportés sur la page Manual Mode.


Procédure d’apprentissage

Cette rubrique explique comment fabriquer un fichier initial de configuration du robot. La page Manual Mode est utilisée.

Chaque axe peut être piloté séparément en choisissant une vitesse et une accélération, et avec des incréments de déplacement compris entre 0,1 et 10 millimètres ou degrés. Le logiciel permet aussi le pilotage du TCP directement en cartésien, c’est-à-dire en agissant sur les coordonnées x, y et z.

Ces outils permettent de définir les points nécessaires au processus.


Le préhenseur peut aussi être manipulé par des commandes spécifiques : rotation de plus ou moins 90°, et ouverture/fermeture de la pince. Le réglage d’écartement des doigts n’est réglable que par édition du fichier de configuration (paramètres GripperOpen et GripperClose).


Pour mémoriser les coordonnées d’un point, il suffit de sélectionner son nom dans la liste puis de cliquer sur le bouton « Update ».

Le bouton « Goto » permet d’ordonner au robot de rejoindre soit la position saisie juste au-dessus, avec une vitesse linéaire du TCP modifiable, soit la position définie par un point de la liste.

Dans ce dernier cas, la position verticale visée est minorée par une marge de sécurité. Ce paramètre, nommé ZEscape, n’est réglable que par édition du fichier de configuration.

Les commandes « Down » et « Up » sont réglées pour utiliser ce paramètre.


Le fichier de configuration peut être sauvegardé par la commande « Save ».
